# Introduction to whecho

whecho is a CLI tool that brings webhook notifications to your command line. Think of it as "Linux echo with webhooks" – it lets you send messages to various webhook platforms directly from your terminal or Python scripts.

## Why whecho?

Don't guess when a job is finished – have it message you! Whether you're running long scripts, monitoring system tasks, or automating workflows, whecho keeps you informed by sending notifications to your favorite messaging platforms.

## Key Benefits

- **Simple command-line interface**: Send notifications with a single command
- **Python integration**: Use whecho in your Python scripts with one import
- **Multiple platforms**: Support for Discord, Slack, Webex, and Microsoft Teams
- **Cross-platform**: Works on Linux, Windows, and macOS
- **Lightweight**: Minimal dependencies, quick installation
- **Configurable**: Set default webhooks and customize user/machine identifiers

## Supported Platforms

whecho supports webhooks for the following platforms:

- **Discord** – Send messages to Discord channels
- **Slack** – Post to Slack workspaces
- **Webex** – Send markdown-formatted messages to Webex Teams
- **Microsoft Teams** – Notify Teams channels
- **Generic webhooks** – Works with any webhook expecting JSON with a text field

## Use Cases

- **Long-running scripts**: Get notified when batch jobs, data processing, or builds complete
- **System monitoring**: Alert yourself when system events occur
- **CI/CD pipelines**: Send build status notifications to your team
- **Automation workflows**: Integrate notifications into cron jobs and scheduled tasks
- **Development debugging**: Quick notifications from Python scripts during development
