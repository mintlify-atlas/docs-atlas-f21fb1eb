Get up and running with whecho in just a few steps. This guide will walk you through installation, configuration, and sending your first webhook notification.

<Steps>
  <Step title="Install whecho">
    Install whecho using pip:

    ```bash
    pip install whecho
    ```

    Verify the installation:

    ```bash
    whecho --version
    ```
  </Step>

  <Step title="Get a webhook URL">
    Choose a platform and create a webhook URL. whecho supports:

    <CardGroup cols={2}>
      <Card title="Discord" icon="discord" href="https://support.discord.com/hc/en-us/articles/228383668-Intro-to-Webhooks">
        Server Settings → Integrations → Webhooks → New Webhook
      </Card>
      <Card title="Slack" icon="slack" href="https://api.slack.com/messaging/webhooks">
        Create an incoming webhook for your workspace
      </Card>
      <Card title="Webex" icon="cisco" href="https://apphub.webex.com/applications/incoming-webhooks-cisco-systems-38054-23307-75252">
        Add incoming webhook to your Webex space
      </Card>
      <Card title="Microsoft Teams" icon="microsoft" href="https://docs.microsoft.com/en-us/microsoftteams/platform/webhooks-and-connectors/how-to/add-incoming-webhook">
        Add incoming webhook connector to your channel
      </Card>
    </CardGroup>

    <Note>
      Keep your webhook URL secure - anyone with access to it can send messages to your channel.
    </Note>
  </Step>

  <Step title="Initialize configuration">
    Run the initialization command to set up your default webhook URL:

    ```bash
    whecho --init
    ```

    You'll see an interactive configuration menu:

    ```bash Example configuration
    Current config:
    [1] default_url: None
    [2] user: craut
    [3] machine: craut-spectre

    Please enter the number/name of the config option you would like to modify (empty or Q to exit): 1
    Please enter the new value for default_url: https://discord.com/api/webhooks/...
    Successfully modified default_url to https://discord.com/api/webhooks/...!
    Current config:
    [1] default_url: https://discord.com/api/webhooks/...
    [2] user: craut
    [3] machine: craut-spectre

    Please enter the number/name of the config option you would like to modify (empty or Q to exit): q
    Successfully initialized whecho!
    ```

    <Info>
      The `user` and `machine` values are automatically detected from your system and included in Discord notifications. You can modify them if desired.
    </Info>
  </Step>

  <Step title="Send your first message">
    Now you're ready to send your first webhook notification:

    ```bash
    whecho "Hello from whecho!"
    ```

    Your message will be sent to the webhook URL you configured. Check your Discord/Slack/Webex channel to see it appear!
  </Step>
</Steps>

## Usage Examples

### Basic message
```bash
whecho "Build completed successfully"
```

### Using a different webhook URL
```bash
whecho -u https://hooks.slack.com/services/YOUR/WEBHOOK/URL "Slack message"
```

### Debug mode
```bash
whecho -d "Testing with debug output"
```

### Multi-word messages
```bash
# These are equivalent:
whecho hello world
whecho "hello world"
whecho -m "hello world"
```

## Using whecho with Python

You can also use whecho directly in your Python scripts:

```python
from whecho.whecho import whecho_simple

# Use the default configured webhook URL
whecho_simple("Script completed successfully!")

# Use a custom webhook URL
whecho_simple("Alert: High CPU usage", url="https://discord.com/api/webhooks/...")

# Enable debug mode
whecho_simple("Debug test", debug=True)
```

## Common Use Cases

<AccordionGroup>
  <Accordion title="Notify when long-running scripts complete">
    ```bash
    python train_model.py && whecho "Model training complete!"
    ```
  </Accordion>

  <Accordion title="Alert on errors">
    ```bash
    ./deploy.sh || whecho "Deployment failed!"
    ```
  </Accordion>

  <Accordion title="Cron job notifications">
    ```bash
    0 2 * * * /path/to/backup.sh && whecho "Nightly backup completed"
    ```
  </Accordion>

  <Accordion title="Python script integration">
    ```python
    from whecho.whecho import whecho_simple

    def process_data():
        try:
            # ... your processing logic ...
            whecho_simple("Data processing completed successfully")
        except Exception as e:
            whecho_simple(f"Data processing failed: {str(e)}")
    ```
  </Accordion>
</AccordionGroup>
